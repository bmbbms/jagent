# 整体实施分解

## 实施总目标

第一阶段不是把全部能力做完，而是完成三件事：

1. 搭出可复用的平台骨架
2. 跑通三个代表性业务闭环
3. 建立后续可持续扩展的接入规范

建议第一阶段目标定义为：

- 商户AI 跑通知识问答和问题处理
- 运营支持 跑通调额、入网、变更审核辅助
- 数据支持 跑通销售数据查询和固定报表辅助
- 同时具备权限、审批、审计、知识检索、Tool接入、基础评测

## 双线推进思路

- 平台底座线：权限、审计、审批、知识、工具、观测
- 业务场景线：商户AI、运营支持、数据支持

## 八个工作流

### 1. 业务梳理与规则沉淀

主要任务：

- 梳理四大业务域和一期范围
- 明确每个场景的输入、输出、处理路径
- 梳理审核规则、升级人工规则、审批边界
- 统一业务口径、字段口径、结果口径
- 产出首批 Skill 清单和 Workflow 清单

交付物：

- 场景清单
- Skill 清单
- Workflow 清单
- Tool 清单
- 业务规则文档

### 2. 平台架构与工程骨架

主要任务：

- 初始化 Agno + FastAPI 主工程
- 建立项目目录规范
- 搭建 Router Agent 和 Domain Agent 骨架
- 建立 Skill、Tool、Workflow、Knowledge 的基础注册机制
- 设计统一配置、日志、异常处理机制

交付物：

- 项目主仓骨架
- 基础目录结构
- Agent 注册骨架
- 配置与环境规范

### 3. 权限、审批、审计底座

主要任务：

- 接入统一登录
- 建立 RBAC
- 设计数据权限和知识权限传递
- 建立审批任务模型
- 建立工具调用审计、结果审计、审批审计

交付物：

- 用户、角色、权限模型
- 审批流基础能力
- 审计日志能力
- 高风险工具调用规范

### 4. Tool / MCP 接入层

主要任务：

- 梳理一期需要接的系统
- 定义 Tool 接口规范
- 把只读查询和写操作分级
- 标记高风险工具是否需要审批
- 统一异常返回、超时、重试、限流机制

一期优先 Tool：

- 商户资料查询
- 交易概况查询
- 风险标签查询
- 审批提交流程
- 数据报表查询
- 工单/问题单提交

交付物：

- Tool Registry
- 核心 Tool SDK
- MCP / 内部 API 适配层

### 5. Knowledge / RAG 能力

主要任务：

- 梳理一期知识源
- 建立知识接入、清洗、切片、索引流程
- 做知识标签、权限标签、版本标签
- 建立检索与引用返回机制
- 建立知识更新流程

一期优先知识源：

- 商户 FAQ
- 入网、变更、调额制度
- 合规规则
- 清算、差错处理手册
- 数据口径文档

交付物：

- 知识接入流程
- 知识库索引
- 权限过滤方案
- 引用返回规范

### 6. Skill 体系建设

主要任务：

- 统一 SKILL.md 模板
- 首批 Skills 编写和评审
- 为每个 Skill 定义适用场景、输入、步骤、输出、升级条件
- 将 Skill 与 Tool、Workflow 绑定
- 建立 Skill 版本管理和启停机制

一期优先 Skills：

- merchant_qa
- merchant_issue_handling
- quota_review
- merchant_onboarding_review
- merchant_change_review
- direct_sales_data_assistant

交付物：

- Skills 仓库
- 首批 Skills
- Skill 治理规范

### 7. Workflow 场景建设

主要任务：

- 定义调额审核流程
- 定义入网审核流程
- 定义变更审核流程
- 明确每一步输入、规则校验、审批节点、异常回退
- 接入审批系统和留痕

交付物：

- Workflow 模型
- 一期 2 到 3 条关键审核流程
- 人工介入与回退机制

### 8. 前端与运营看板

主要任务：

- 搭建工作台
- 做对话区、引用区、操作区、审批区
- 展示工具调用结果和知识依据
- 展示审批状态和处理记录
- 建立基础运营看板

交付物：

- Web 工作台
- 审批操作页
- 基础指标看板

## 分阶段里程碑

### 阶段 0：启动与对齐，1 到 2 周

- 确认一期范围
- 梳理首批场景
- 梳理系统接入清单
- 定 Skill、Tool、Workflow 模板
- 定项目组织和节奏

### 阶段 1：平台骨架，2 到 4 周

- 主工程初始化
- 登录、权限、审计基础能力
- Tool 接入框架
- Knowledge 接入框架
- Skill Registry / Tool Registry 基础版

### 阶段 2：一期 MVP 场景，4 到 6 周

- 商户AI 问答和问题处理
- 运营支持审核辅助
- 数据支持查询助手
- 首批 Skills 和 Workflows 落地

### 阶段 3：试点上线与优化，2 到 4 周

- 试点用户接入
- 问题修复和策略调优
- 评测与反馈闭环
- 监控、告警、审批流优化

## 一期优先级

### P0 必做

- 项目骨架
- 登录权限
- 审计日志
- Tool 接入框架
- Knowledge 检索框架
- 商户AI 问答
- 调额审核辅助
- 数据查询助手

### P1 应做

- 入网审核辅助
- 变更审核辅助
- 审批流联动
- Skill Registry
- 基础评测

### P2 后做

- 代理商AI
- 更多运营子场景
- 更复杂的多 Agent 协同
- 更强分析型报表能力

## 团队角色建议

- 产品/业务负责人
- 架构/后端负责人
- AI/Agent负责人
- 前端负责人
- 数据/知识负责人
- 业务专家

## 关键实施风险

- 范围失控
- 业务规则不清
- 系统接入过慢
- 权限和审计滞后
- 过早追求智能化

## 立刻启动的第一批动作

1. 冻结一期业务范围
2. 拉出首批 10 到 15 个核心场景
3. 列出首批知识源和系统接入清单
4. 定义 Skill / Tool / Workflow 模板
5. 定义高风险动作和审批边界
6. 确定项目主仓和工程骨架
7. 明确试点部门和试点用户
