# 第一期建设范围、边界与路线图

## 第一期建设原则

第一期的目标不是一次做全，而是先把“平台底座 + 代表性场景 + 风险闭环”跑通。

## 第一期建设范围

### 商户AI

- 知识库问答
- 常见问题解答
- 问题分类与处理建议
- 基于知识和案例的标准答复

### 运营支持

- 调额审核辅助
- 入网审核辅助
- 变更审核辅助
- 审核意见生成
- 人工审批流转与留痕

### 数据支持

- 直营销售数据助手
- 基础经营指标查询
- 固定报表辅助生成
- 合规监管报表辅助

## 第一期必须建设的平台能力

- 统一登录与权限控制
- Skill Registry
- Tool Registry
- Workflow + Approval
- Knowledge 检索
- 审计日志
- 基础监控与评测

## 第一期明确不做

- 不追求全业务域一次性覆盖
- 不做完全自治的多 Agent 协作
- 不做大规模自动写操作闭环
- 不做过于开放的自由数据分析
- 不把所有运营审核流程都自动化
- 不在第一期做复杂跨部门联动编排

## 第一期成功标准

- 能稳定支撑三类核心场景
- 能完成权限控制、审批流、审计留痕
- 能让知识问答、审核辅助、数据查询形成闭环
- 能支撑后续按模块扩展新场景

## 三阶段演进路线

### 阶段一：MVP验证

目标：

- 验证核心业务价值
- 搭建平台底座
- 跑通代表性闭环场景

重点建设：

- 商户AI问答
- 运营审核辅助
- 数据支持助手
- 权限、审批、审计、知识库

关键词：

- 先可用
- 先闭环
- 先安全

### 阶段二：平台扩展

目标：

- 从单点场景走向多业务域复用
- 建立标准接入机制
- 提升治理能力

重点建设：

- 接入代理商AI
- 扩展更多运营支持场景
- 增加更多技能、工具、流程模板
- 完善技能版本管理、工具分级、流程治理、评测体系

关键词：

- 可复用
- 可扩展
- 可治理

### 阶段三：智能运营平台

目标：

- 从“几个 AI 能力”演进成“统一智能服务平台”
- 让新场景接入更快
- 让运营和管理层能持续看到价值

重点建设：

- 跨业务域协同
- 更强的数据分析与经营洞察
- 场景配置化接入
- 精细化评测和持续优化
- 平台级运营看板

关键词：

- 平台化
- 体系化
- 持续优化
